L'utilisateur souhaite construire un dashboard de monitoring personnel pour deux ressources clés de son environnement de développement.

**Périmètre:**
- Consommation de tokens Claude (Anthropic API)
- Temps de Codespace GitHub restant (GitHub API)
- Affichage simple, pas d'interactivité requise dans un premier temps
- Données rafraîchies automatiquement (polling ou temps réel)

**Why:** L'utilisateur développe activement dans Claude Code sur Codespace et veut surveiller sa consommation quotidienne de ressources pour éviter les dépassements ou coupures inattendues.

**How to apply:** Proposer des solutions légères et autonomes, facilement déployables dans un environnement Codespace. Éviter les architectures lourdes ou over-engineered. Privilégier la lisibilité et la maintenance simple pour un usage personnel.
