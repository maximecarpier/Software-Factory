**Choix rejetés :** aucun rejet — style clair/iOS épuré accepté dès la sélection (option B).
Le dark mode était proposé mais l'utilisateur a explicitement choisi la clarté iOS.

**Patterns validés :**
- Bottom tab bar + FAB : pattern iOS standard, adopté sans friction
- Left-border stripe colorée pour la priorité (rouge/orange/gris) : visuellement distinct, pas de texte de priorité nécessaire sur la carte
- Chips multi-sélection horizontaux scrollables pour les filtres : remplace les selects desktop
- Toggle buttons pour Type/Priorité dans le formulaire : remplace les `<select>` peu ergonomiques au pouce
- Swipe gesture (droite=terminé, gauche=éditer) : pattern iOS natif, zone de reveal colorée derrière la carte

**Points d'attention pour prochaines sessions :**
- `navigator.vibrate()` absent iOS Safari — le haptic est always best-effort sur la cible principale
- Pull-to-refresh manuel sur iOS PWA = risque de conflits avec scroll natif → à tester impérativement avant Gate 3
- Le brief utilisateur étendait le périmètre des specs v2.0 (suppression, statut, vue Projets, recherche) sans passer par specs-framer — signalé dans inter-agent.md mais à surveiller
- La refonte a ajouté des fonctionnalités : toujours vérifier si le brief de refonte UI implique une montée en version des specs (v3.0 ici)

**À améliorer :**
- Demander en Phase Zéro si des contraintes d'accessibilité existent (contrast ratio WCAG, VoiceOver)
- La spécification des icônes SVG est absente — le designer devrait fournir les SVG ou des descriptions précises (chevron, pencil, check) pour que code-implementer ne choisisse pas n'importe quoi
